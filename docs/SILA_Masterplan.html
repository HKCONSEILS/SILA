1. Vision et périmètre

1.1 Objectif

Construire un pipeline self-hosted, modulaire et industrialisable, capable de prendre en entrée une vidéo longue (jusqu'à 1h+), d'en produire des versions doublées dans plusieurs langues cibles, avec une voix clonée cohérente, un alignement temporel contrôlé, et un rendu audio crédible.

1.2 Ce que le système fait

  • Extraire l'audio d'une vidéo source
  • Séparer la voix du fond sonore (musique, ambiance, SFX)
  • Transcrire la parole avec timestamps mot-à-mot et identification des locuteurs
  • Segmenter le contenu parlé en unités de traduction/TTS
  • Traduire chaque segment vers N langues cibles avec contexte et glossaire
  • Réécrire les traductions trop longues pour respecter le timing original
  • Cloner la voix et générer l'audio cible par segment et par langue
  • Ajuster la durée de chaque segment TTS pour respecter le slot temporel
  • Mixer l'audio TTS avec le fond sonore, normaliser le loudness
  • Remuxer l'audio final avec la piste vidéo source (sans réencodage)
  • Exporter un fichier MP4 par langue cible

1.3 Ce que le système ne fait pas

  • Temps réel ou streaming live
  • Traitement vidéo (réencodage, modification des frames)
  • Sous-titrage seul (c'est un sous-produit)
  • Qualité "studio broadcast" universelle
  • Lip-sync en V1-V2

1.4 Cadre par défaut

  • Self-hosted, local ou infra privée
  • Vidéos longues (1h+) comme cas normal
  • Pipeline relançable, batchable, parallélisable et auditable
  • Approche réaliste et industrialisable, pas démo théorique
  • Pas de SaaS managé sauf nécessité démontrée

2.1 Principes architecturaux verrouillés

3.3 Détail des briques TTS

CosyVoice 3.0 (principal)

  • Modèle : Fun-CosyVoice3-0.5B-2512
  • Langues : 9 (zh, en, ja, ko, de, es, fr, it, ru) + 18 dialectes chinois
  • Features : Zero-shot cross-lingual cloning, pronunciation inpainting, instruction-based control, streaming
  • Taux d'erreur : CER 0.81%, speaker similarity 77.4%
  • Sample rate : 22.05 kHz interne → resample 48 kHz en sortie

Qwen3-TTS (alternatif)

  • Modèle : Qwen3-TTS-12Hz-1.7B-Base + CustomVoice
  • Langues : 10 (zh, en, ja, ko, de, fr, ru, pt, es, it)
  • Features : Clonage 3s de référence, voice design textuel, génération longue (10 min)
  • Taux d'erreur : WER 1.835%, speaker similarity 0.789

3.4 LLM de réécriture (V2)

Mistral Small 3.2 24B (principal)

  • Quantification : Unsloth Dynamic 2.0 Q4_K_M (~15 Go VRAM)
  • Multilingue 40+ langues, function calling, JSON output, context 128k

Ministral 3 8B (fallback)

  • ~5 Go VRAM. Usage : quand la VRAM est occupée par le TTS.

6. Workflow détaillé — DAG du pipeline

6.1 Vue d'ensemble du DAG

0
Phase 0 : INGEST
Créer projet, uploader vidéo, ffprobe, écrire manifeste initial
1
Phase 1 : EXTRACTION
FFmpeg → audio WAV 48kHz mono + métadonnées vidéo
2
Phase 2 : SÉPARATION VOCALE V2+
Demucs v4 → stems : voice.wav, music.wav, sfx.wav
3
Phase 3 : ANALYSE SPEECH
WhisperX ASR → transcript brut + alignement mot-à-mot + diarisation [V2+]
4
Phase 4 : SEGMENTATION LOGIQUE
Réconciliation transcript, attribution speaker IDs, segmentation métier (§6.2), attribution timing_budget_ms
5
Phase 5 : CONTEXTE GLOBAL V2+
Résumé narratif, extraction entités, glossaire projet, profils voix
6
Phase 6 : TRADUCTION
NLLB-200 segment par segment, fenêtre glissante 2-3 segments, glossaire + contexte. Classer timing fit.
7
Phase 7 : RÉÉCRITURE CONTRAINTE V2+
Si rewrite_needed : LLM local → variante courte respectant timing_budget_ms
8
Phase 8 : TTS / VOICE CLONING
CosyVoice 3.0 par segment. Mesurer durée. Time-stretch ≤1.25× si nécessaire.
9
Phase 9 : ASSEMBLY AUDIO
Placer segments sur timeline. Crossfade 50ms. Mixer stems [V2+]. Loudnorm -16 LUFS.
10
Phase 10 : QC AUTOMATIQUE
Vérifier trous, clipping, true peak, loudness, timing fit. Tagger segments problématiques.
11
Phase 11 : EXPORT
FFmpeg remux (vidéo copy + audio AAC). Générer SRT/VTT. Rapport QC JSON.
12
Phase 12 : LIP-SYNC V3
Uniquement sur segments face-caméra avec désync visible.

6.2 Règles de segmentation logique

#RègleDétail
S1Pas de mélange de locuteursUn segment = un seul speaker_id
S2Couper sur pause + ponctuationPause > 400ms + ponctuation forte (. ? !)
S3Durée nominale 4-8sCœur de distribution visé
S4Hard cap 10sAu-delà, forcer coupe sur ponctuation faible ou pause > 200ms
S5Minimum 3sEn dessous, fusionner avec segment adjacent
S6Seuil adaptatifSi contenu dense, réduire le seuil à 300ms
S7OverlapsFlag overlap: true, traitement dédié
S8Contexte conservécontext_left (2-3 seg.) + context_right (1-2 seg.)

12. Gestion d'erreurs et reprise

12.1 Clé d'idempotence

  project_id + stage_name + target_lang +
  segment_id + model_name + model_version +
  params_hash + input_artifact_hash

12.2 Politique de retry

CasAction
OOM GPU, timeout, erreur stockageRetry 3× backoff exponentiel (2s, 8s, 32s)
Worker crashCelery re-dispatch automatique
Langue non supportée TTSFallback moteur alternatif ou hard-fail avec flag
Voice profile manquantFlag review_required + fallback voix générique
Timing impossible (stretch > 1.25×)Flag review_required, garder stretché à 1.25×
Segment pathologiqueQuarantaine + flag manual_review
3 échecs consécutifsDead letter → statut failed

12.3 Granularité de relance (V1)

--retry segment seg_0042 --lang en
--retry failed --lang en
--retry from-stage translate --lang en
--retry stage assembly --lang en

12.4 Validation avant export

  • < 5% en failed → proposer export partiel (segments failed = silence)
  • > 5% en failed → bloquer l'export, signaler à l'opérateur

17. Glossaire

TermeDéfinition
Timing contractEngagement de durée maximale d'un segment, fixé dès la segmentation.
Timing budgetDurée en ms allouée à un segment pour le TTS + stretch.
Cascade de duréeStratégie multi-étapes pour respecter le timing.
Tronc communÉtapes exécutées une seule fois quel que soit le nombre de langues.
Segment logiqueUnité de traduction/TTS (3-10s, mono-speaker).
Chunk techniqueDécoupage physique pour le calcul distribué (30-120s avec overlap).
StemPiste audio séparée produite par Demucs.
Golden setVidéos de référence pour benchmarks.
Quality gateCondition à remplir avant l'étape suivante.
Dead letterSegment ayant échoué 3× → quarantaine.
Fan-outPoint du DAG où le pipeline se ramifie en N branches.
IdempotenceOpération qui, exécutée plusieurs fois, produit le même résultat.
DuckingRéduction automatique du volume de la musique sous la voix.
LUFSLoudness Units relative to Full Scale — mesure du loudness perçu.
CERCharacter Error Rate — taux d'erreur au caractère (TTS).
WERWord Error Rate — taux d'erreur au mot (ASR).

18. Historique des décisions (ADR)

ADR-001 : CosyVoice 3.0 plutôt que XTTS v2

CosyVoice 3.0 (Apache 2.0) retenu pour le TTS multilingue avec voice cloning. XTTS v2 rejeté (CPML non-commercial, Coqui fermé). 9 langues couvertes nativement.

ADR-002 : NLLB-200 avec risque licence

NLLB-200 3.3B retenu en V1 (usage interne). MADLAD-400 préparé comme alternative pour usage commercial.

ADR-003 : Celery + Redis plutôt que Temporal V1-V2

V1 = script séquentiel. V2 = Celery + Redis. V3 = Temporal. Complexité opérationnelle minimale au démarrage.

ADR-004 : Mistral Small 3.2 24B pour la réécriture

Mistral Small 3.2 24B (Q4_K_M, ~15 Go VRAM) retenu V2. Apache 2.0, 40+ langues, tient sur 1× RTX 4090. Fallback Ministral 3 8B ~5 Go.

ADR-005 : Pas de Demucs en V1

V1 cible contenus voix-only (podcasts, cours). Demucs ajouté en V2 pour les contenus avec musique/SFX.
